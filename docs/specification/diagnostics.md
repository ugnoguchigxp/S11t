# S11tnext diagnostics

CLI diagnostics are stable objects with `code`, `severity`, `message`, `file`, and `path`; TOML syntax
errors also include line and column. Human output is written to stderr. `--format json` emits a JSON
array for machine consumers.

The main categories are:

- configuration and IO: `S11TNEXT_CONFIG_INVALID`, `S11TNEXT_FILE_NOT_FOUND`,
  `S11TNEXT_SOURCE_DIR_NOT_FOUND`, `S11TNEXT_SOURCE_EMPTY`, `S11TNEXT_TOML_SYNTAX`;
- authoring: `S11TNEXT_SOURCE_INVALID`, `S11TNEXT_SOURCE_SHAPE_CONFLICT`, `S11TNEXT_KEY_INVALID`,
  `S11TNEXT_KEY_COLLISION`, `S11TNEXT_OWNER_UNRESOLVED`;
- locale and policy: `S11TNEXT_RELEASE_PROFILE_REQUIRED`, `S11TNEXT_RELEASE_PROFILE_NOT_FOUND`,
  `S11TNEXT_TRANSLATION_MISSING`, `S11TNEXT_TRANSLATION_PLACEHOLDER_MISMATCH`,
  `S11TNEXT_LOCALE_INVALID`, `S11TNEXT_LOCALE_NOT_FOUND`;
- variables and safety: `S11TNEXT_VARIABLE_PROFILE_NOT_FOUND`, `S11TNEXT_VARIABLE_UNDECLARED`,
  `S11TNEXT_VARIABLE_UNUSED`, `S11TNEXT_PLACEHOLDER_INVALID`, `S11TNEXT_ENCODING_TYPE_MISMATCH`,
  `S11TNEXT_UNSAFE_UNTRUSTED_RAW`, `S11TNEXT_UNSAFE_UNTRUSTED_PLACEMENT`;
- generated output: `S11TNEXT_BUILD_STALE`;
- runtime: `S11TNEXT_ARTIFACT_INVALID`, `S11TNEXT_ARTIFACT_DIGEST_MISMATCH`,
  `S11TNEXT_CONTEXT_NOT_FOUND`, `S11TNEXT_VALUE_MISSING`, `S11TNEXT_VALUE_EXTRA`, `S11TNEXT_VALUE_INVALID`.

CLI exit codes are `0` for success, `1` for validation failure, `2` for command misuse, and `3` for an
unexpected internal error.
