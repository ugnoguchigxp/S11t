# Security Policy

Security fixes are provided for the latest published S11tnext release line. Before the first stable release,
reports against `main` are also accepted. Please report suspected vulnerabilities through
[GitHub private vulnerability reporting](https://github.com/ugnoguchigxp/s11tnext/security/advisories/new)
rather than opening a public issue. Include affected versions, reproduction steps, and potential impact.

Security fixes will be coordinated before disclosure. SystemContext encoding is not an authorization or prompt-injection security boundary; host applications remain responsible for authorization and tool enforcement.
